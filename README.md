单线程下载知识点回顾:
	1,Activity--->Service传输数据
	2,在Service中开启子线程进行下载
	3,将下来到的文件长度发送给Handler进行处理
	4,在Service中的Handler中去实现下载的功能
	5,判断是否是第一次下载，从线程数据库中查询，如果是下载暂停就接着上一次下载，不是则从头开始下载
	6,在下载线程中，我们跟踪我们的下载进度，并通过广播实时的发送给我们的Activity来更新进度条
	7,在下载完成的时候删除我们的线程信息(1,如果我们一次下载完(没有暂停)，则删除当前线程(其实是空的)。若我们暂停后再继续下载完成，我们在暂停的时候去更新我们的
		线程信息，然后在继续下载的时候去获取这些信息，下载完成后删除该线程的信息)。
	8,这里我们用到了一个很特别的java类RandomAccessFile,该类直接继承Object类，用来实现我们的断点续传功能。
	9,我们这里必须把我们的文件类进行序列化，不然Activity------->>Service是如法获取到数据的。
	10,我们这里对我们的线程信息是通过ORMLitem框架来进行管理的，这样我们可以很方便的去进行CURD了，也方便我们去实现各种secelt的更复杂的查询条件。
	11,只要是耗时的操作我们最后是放在Service里面去进行，第一是因为为了降低我们Activity的负荷，避免出现ANR事件，第二是因为我们的Service的权限比较高，不容易被系统回收。
